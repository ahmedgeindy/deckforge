## 7. AI-TELL BLACKLIST (exhaustive — powers the elimination pass)

Root cause of all tells: generation per-slide in isolation, layout chosen by content SHAPE not rhetorical JOB, output regressing to the training median — uniformity where meaning should drive variation [Tools-B, Tells]. Master test: did the content choose this, or the template?

Format: **Tell — WHY it reads machine-made — DETECT heuristic → FIX (human-craft alternative).**

### Layout repetition
1. **Same archetype on consecutive slides** — humans plan a deck as a narrative with pacing; a run of identical archetypes reads as a loop, not a story (slide-to-slide narrative is precisely where AI is weakest) — DETECT: run ≥3 identical archetypes; any archetype >40% of content slides → FIX: vary archetype when the content's ROLE changes (context→evidence→implication→decision); deliberate echo only for genuinely parallel content, where the echo maps to a parallel in meaning [Tells-A1].
2. **Equal-card walls** — equal cells assert all items are equally important, a claim almost never true of real content; humans rank, machines enumerate; repetition shows layout chosen by item COUNT — DETECT: ≥3 slides with N≥3 cards, all widths within ~5%; constant card count across slides → FIX: rank items — lead item 2–3× area or promoted out of the row, rest demoted to a quiet list; or replace the grid with the diagram it was hiding; cards only for genuinely parallel countable items, once or twice per deck [Tells-A2, Tools-B1, C3].
3. **Icon+title+2-lines ×N grids** — mechanical repetition of one component; stock icons are one-to-many symbols (a rocket "means" growth/launch/speed/startup) so the slide carries zero information — DETECT: ≥4 sibling blocks of small-icon-over-short-title-over-≤2-lines; lightbulb/rocket/gear/handshake vocabulary → FIX: replace dominant cells with real evidence; one concrete proof beats six abstractions [Tells-A3].
4. **Three-column everything** — three-up is the template default for "balanced"; recurring regardless of whether content has 2, 4, or 7 natural parts reveals content poured into a mold — DETECT: >30% of content slides on 3 equal columns; non-parallel content types in equal cells → FIX: let cardinality and asymmetry pick the split (2/3–1/3 when one element is the point; full-bleed when one image IS the slide); column count changes because content structure changes [Tells-A4].
5. **Repeated hero metrics** — the big-number archetype works via SCARCITY; repetition flattens the intensity curve and signals per-slide generation with no memory of the previous slide — DETECT: ≥3 slides dominated by a large numeral in same position/size/color; repeated stat-banner rows → FIX: budget 1–2 hero treatments per deck at the argument's peak; other numbers inline or charted against a reference value [Tells-A5, Tools-B2].
6. **Evenly-stretched sparse content** — AI treats the canvas as a container to fill; "mathematically perfect balance" reads sterile and rigid; humans cluster whitespace as a signal of confidence and focus — DETECT: ≤3 elements at near-equal intervals spanning the canvas; no element ≥1.5× the visual weight of others; margins ≈ inter-element gaps → FIX: dominant element ≥2× weight placed on the reading path; whitespace clustered, not rationed [Tells-A6].
7. **Identical consecutive layouts / no visual arc** — humans naturally vary composition with the narrative (dense evidence → breathing-room statement); uniform structure = no deck-level state — DETECT: same margins, text widths, spacing ratios throughout; slide 12 structurally = slide 3 → FIX: design rhythm as a sequence — alternate density, alternate alignment, insert near-empty slides after heavy ones [Tools-B3].
8. **Decorative title dividers + uniform title format** — the positional anchor is good craft; the tell is the decorative layer (same accent rule under every title) plus syntactic templating ("Six Words, With Colon") — chrome repeated with zero informational role — DETECT: identical rule under 100% of titles; >80% titles matching one syntactic pattern; every title within ±1 word of the same length → FIX: keep the anchor, delete the divider (hierarchy from size/weight/space); vary title FORM by function — question/verdict/bare number [Tells-A7, B2, C4].

### Decoration
9. **Container gradients (esp. purple/blue)** — indigo/violet defaults dominate the training data; gradient = polish applied indiscriminately, decoration substituting for hierarchy; viewers now associate soft blue-violet container gradients specifically with AI output — DETECT: gradients on backgrounds AND cards AND buttons; accent hue ~260–290° with no brand rationale; gradient text → FIX: flat confident color derived from brand or subject; at most one intentional deck-signature gradient (e.g., dividers only), each answering "what does this separate?" [Tells-B1, Tools-B4].
10. **Pill/badge/tag overuse** — a web-UI interactive affordance pasted into a static medium; promises interactivity that doesn't exist and fragments the glance into small rounded containers — DETECT: pill label above titles on >2 slides; >4 pills on any slide; pills repeating the title's words → FIX: typographic kickers (small caps, letter-spaced); one container style per slide max; any filled container reserved for the one element that must pop [Tells-B3, Tools-B7].
11. **Colored left-border cards** — a blockquote CSS idiom leaked into everything; "almost as reliable a sign of AI-generated design as em-dashes" — DETECT: 3–4px colored left stripe on cards/quotes/callouts, same treatment on multiple content types → FIX: distinct treatment per semantic type — quotes by typography (larger, attribution), warnings by icon+color, data by charts; one visual idiom per semantic type [Tells-B4].
12. **Uniform generous radius on everything** — one CSS token applied globally; humans vary radius with element size and role, or use none — DETECT: every rect shares one radius; ≥16px on small chips AND large panels alike → FIX: radius scale tied to element size, or commit to sharp corners for an editorial feel; un-card photos and charts entirely [Tells-B5].
13. **Colored glows / oversized soft shadows** — depth attempted through effects rather than layout hierarchy; on projection, glow reads as blur and shadows imply stacking that doesn't exist — DETECT: brand-colored box-shadows; blur radii >~30px; glow on text → FIX: flat or near-flat; separation via background tone steps and spacing; one subtle neutral shadow level if needed [Tells-B6].
14. **Decorative lines and frame furniture** — separators are needed only when spacing fails to group; a generator that can't trust its spacing adds lines; a designer who controls spacing doesn't need them — DETECT: rules under every title, corner accents, side bars, dot grids, floating shapes with no function → FIX: whitespace and type-scale contrast do the separating; every line must resolve an actual grouping ambiguity [Tools-B6, Tells-B2].

### Content shape
15. **Centered-text overload** — centering is the zero-decision layout: it avoids committing to a reading direction or hierarchy; centered multi-line text destroys the left scan edge — DETECT: >50% of body blocks center-aligned; any centered block >2 lines → FIX: left-aligned grid with a strong reading edge; centering reserved for single-line statements and dividers where isolation IS the point [Tells-C1, Tools-B8].
16. **Emoji as bullets/icons** — reads as placeholder/unfinished; renders inconsistently across platforms; injects casual tone regardless of stakes — DETECT: emoji codepoints in list-marker or icon positions; rocket/sparkle/checkmark density → FIX: plain typographic markers or whitespace-separated lines; one drawn icon style at one size if pictographs genuinely aid scanning [Tells-C2].
17. **Identical bullet rhythm** — LLMs output parallel structures: every bullet the same length, verb-first, same tense — the single most reliable text tell; uniform rhythm also implies uniform importance (the same lie as equal cards, in text) — DETECT: 3–4 bullets on >70% of list slides; per-slide word-count CV <0.2 → FIX: cut to the bullets that matter (deleting one from every four-bullet slide is a cheap humanizing pass); vary lengths; promote the key bullet into the assertion title, demote the rest [Tells-C3].
18. **All-caps everywhere** — a default styling treatment; caps everywhere removes the emphasis value caps had, and caps text is slower to read at distance (word shapes lost) — DETECT: caps on section labels AND subtitles AND CTAs AND chart labels simultaneously → FIX: at most one all-caps role in the system (kicker), letter-spaced, small; sentence case elsewhere [Tells-C5].
19. **Notion-doc slides** — flowing-document layout optimizes for READING; slides are GLANCED; doc-first tools produce scroll-shaped thinking on a stage-shaped canvas — DETECT: heading+paragraph+sub-bullets flowing top-down; variable card heights; text-first everything → FIX: one idea composed spatially — focal element, subordinates placed around it, detail exiled to notes/appendix; content that only works as prose belongs in a send-ahead memo [Tools-B9].

### Imagery
20. **Stock 3D / saturated AI imagery / stock metaphors** — abstract saturated renders lack relevance to content; default diffusion output has a recognizable glossy grade; decoration unrelated to the argument is filler the audience must actively ignore — DETECT: imagery with no referent in the slide's text; hyper-gloss teal-orange renders; handshakes/rockets/crystalline shapes; nonsense text inside images → FIX: every image earns its place as evidence (screenshot, real photo, chart) or is the ONE deliberate metaphor of the deck; unify photography with one treatment/grade [Tells-D1, PC §6.1].
21. **Browser-chrome mockups everywhere** — a web-template idiom; repeated chrome adds pixels that say "generic product shot" while shrinking the actual evidence — DETECT: ≥3 slides with traffic-light/URL-bar framing; chrome on cropped details with no window context → FIX: crop to the proving region, enlarged to back-of-room legibility; ≤1 full-window frame per deck, only when "real shipped product" is itself the message (LOW-MED confidence) [Tells-D2].
22. **Generic flat icon sets / mixed icon styles** — default tool libraries pull identical glyph sets the audience has seen in a hundred decks; style-mixing reveals no human curated them — DETECT: line + filled + duotone mixed in one deck; icon-per-bullet decoration → FIX: fewer icons, one style, drawn/configured to the domain; or replace icon vocabulary with tiny real artifacts (mini-chart, logo, photo crop) [Tells-D3].

### Typography & color
23. **Default-font-for-everything** — the ubiquitous UI sans is "the Helvetica of the LLM era": competent, invisible, statistically default — DETECT: one UI sans across all roles at default weights; no size contrast beyond template levels → FIX: type chosen deliberately for room and brand; real display-vs-body contrast via weight/width/pairing [Tells-E1].
24. **Lone serif-italic accent word** — mimics a design-flourish template without full commitment; a single borrowed gesture atop default everything reads as pastiche — DETECT: exactly one italic serif word per title in an otherwise sans system, repeated deck-wide → FIX: commit to a full typographic system; emphasis lands on the operative word, semantically, sparingly [Tells-E2].
25. **Mid-palette "safe" scheme** — AI averages toward muted blues/warm grays, the median of corporate decks; median palette = zero memory encoding — DETECT: palette entirely mid-saturation, mid-lightness; accent used for decoration AND emphasis AND data interchangeably → FIX: commit — brand color at conviction saturation or disciplined near-monochrome; assign the accent ONE job and never spend it decoratively [Tells-E3].

### The inverse failure — fake humanity via random variation
26. **Random variation is equally detectable** — the eye wants familiarity; breaking patterns randomly (fonts mid-deck, palette drift) disorients and weakens the message; variation that doesn't map to a change in meaning is noise, and noise also reads machine-made — DETECT: >2 font families; new hues appearing after slide ~3; title anchor moving without a section change; per-slide radius/shadow changes; archetype run-length of exactly 1 for the whole deck (meaningful parallel content SHOULD echo visually) → FIX: the two-layer rule (§0.6) — system layer invariant deck-wide (grid, margins, type, palette semantics, title anchor, footer identical-or-absent, icon style, image treatment); composition varies BY MEANING, within the grid, via cell spans and density — never off-grid jitter; every variation must answer "what changed in the content to earn this change in form?" [Tells inverse-failure, Ramotion, Toptal].
