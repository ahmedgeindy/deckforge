# PIPELINE POSITION & UPSTREAM CONTRACT

**Pipeline position.** presentation-structure locks the story (assertion titles, beat order, fact bank); presentation-design assigns each beat a rhetorical job and an archetype-level intent, plus all deck-wide tokens, grid, and evidence-craft invariants. THIS skill owns the layer beneath: given a beat whose job is *structure / chronology / experience / flow / decision* and whose content is technical or business prose/bullets, it (a) decides whether a diagram is earned at all, (b) picks the specific FORM the content's relationship licenses, and (c) emits a complete renderable spec for the internal design of that information graphic.

**Contract with upstream (POINT, never restate; never contradict):**

- structure master: G1 assertion titles (every diagram slide still carries a full-sentence headline; the title is written FIRST and the diagram derived from it) · G2 glance test (~3s) · G3 Mayer/CLT (coherence, contiguity, pre-training, ~4-chunk working memory) · G7 builds · G9 dwell budgets · §0 conclusion-placement model · §4 dot-dash/ghost-deck process · F13 slidedoc-vs-stage artifact split · G5 headline honesty.
- design master: §5.2 diagram craft basics (one flow direction; unidirectional verb-labeled arrows; zero crossings; uniform node schema; ≤2 nesting; ~6–12 node cap; builds authored backwards; split by zoom/scenario) · §4.1 archetype specs (Architecture #3, Blueprint #4, Comparison #5, Timeline #7, Journey #8) · §4.1b archetype selection · §5.3 chart-type ladder (QUANTITY relationships route there, not here) · §5.0/Ev§4 focus/defocus recipe · §2.6 grid-span recipes · §6.2 accent/semantic-color discipline · §7 AI-tell blacklist · §1.5 spec-emission contract (this skill's §5 extends it with diagram-specific fields).

**Confidence conventions.** Confidence HIGH unless marked (MED)/(LOW). "Derived" = reasoned from legibility math or upstream invariants where authoritative sources were verified absent — flag in output, never present as citation. Numbers on the 1280×720 basis. Advisor tool was unavailable for all three research sessions and the synthesis (noted for orchestrator).

# 7. CONFLICTS RESOLVED (by rule, never averaged)

| # | Conflict | Positions | Resolution |
|---|---|---|---|
| R1 | Duarte taxonomy versions | slide:ology Flow/Structure/Cluster/Radiate/Pictorial vs Diagrammer Flow/Network/Stack/Segment/Join | Adopt the relationship-verb logic (current, cleaner); Radiate survives as the hub-and-spoke case inside network (1.10) with the delete-the-hub license (§6 #3). |
| R2 | "Why" routing: Roam (multi-variable plot) vs systems thinking (causal loop) | quantitative interaction vs feedback structure | Both kept with a router: why about MAGNITUDES → chart (design §5.3); why about MECHANISMS → CLD (1.7). |
| R3 | Notation fidelity (UML/BPMN/decision codes) vs glance test | standards completeness vs 3-second parse | Mode fork: presented slides strip notation; sent-ahead/technical artifacts restore it (design R21 extended to all diagram notations). |
| R4 | Symmetry vs crossings | aesthetic symmetry vs empirical readability | Resolved for crossings (Purchase's validated ranking); symmetry sacrificed first, and only meaningful symmetry drawn at all (§3.0). |
| R5 | Swimlane necessity | BPMN lanes-by-default vs flowchart minimalism | Lanes ONLY when ownership/handoff is the message (1.3); consistent with "did the content choose this?" |
| R6 | Funnel vs bars | rhetorical shape vs dataviz accuracy | Funnel allowed as sequence-with-numbers when the LOSS is the message; precise stage comparison across segments/periods → bars (design §5.3). Router, not average. |
| R7 | Colored (RAG) Harvey balls vs one-accent discipline | traffic-light convention vs design §6.2 | Monochrome ink balls + single verdict-column accent; RAG only when the matrix IS a status report (then the colors are semantic, not decorative). |
| R8 | Edge bundling "neat vs readable" | visual tidiness vs faithfulness research | Against bundling at slide scale; aggregate to cluster-level edges instead (§3.6). |
| R9 | C4-dynamic "free-form" vs upstream "no free placement" | apparent contradiction | Not a conflict: C4's freedom is from lifeline geometry; on slides it still snaps to the deck grid and reuses the locked architecture layout. |
| R10 | Rule-of-three habit vs honest cardinality | consulting three-up default vs content truth | Cardinality comes from the source content; the rule of three lives at the STORY layer, never as a diagram-layer target (also design Tells-A4). |
| R11 | Diagram enthusiasm vs restraint | Passera/graphic-organizer wins vs SmartArt failure corpus | Not a contradiction: wins occur on content WITH genuine contingency/role/time structure, failures on content without it or with mismatched forms. Encoded as C3 + the ladder §0.2 — the variable is content–form fit, not diagram quantity. |
| R12 | Tables vs flowcharts for contingencies | Wright's results shift with task (MED — via secondary) | Encode task-dependence: lookup-once → flowchart/tree; repeated reference → table; must-be-recalled → spoken track + simplest visual. For decks, optimize for the in-room lookup (presented content leans on narration anyway, Mayer). |
| R13 | 2×2 top-right cliché | consulting practice normalizes; skeptics discount | Kept as a MED warning with mitigations (action-verb quadrants, defensible orthogonal axes, honest placement) rather than a ban — the form itself is the experts' most-preferred comparison format. |
| R14 | Meyer's 5 text structures vs the 13-row cue table | taxonomy granularity | Not a conflict: extra rows are presentation-specific refinements (chronology split from sequence; swimlane split by actor-multiplicity), each mapping back to a Meyer class, subdivided where the target VISUAL differs. |
| R15 | Derived caps (lanes 3–5, lifelines 3–5, yes/no sides, node counts) | practitioner folklore vs verified absence of authoritative sources | All flagged DERIVED from legibility math + upstream glance-test invariants; presented as bands, never citations; replaced by consistency rules where no number is defensible (§3.3). Do not cite these numbers to users as external authority. |
| R16 | Effect-size generalization | Passera single-domain; Guo covers reading comprehension broadly | Directionally consistent; used as design targets (speed gate), never quoted as deck-specific guarantees (MED). |
